# HireNow Bot

Telegram-бот для пошуку роботи в IT: кандидати можуть створити анкету, а роботодавці - оформити вакансію та шукати релевантні збіги за навичками, досвідом і локацією.

## Що вже є

- роль `Працівник` з анкетою, навичками, досвідом, локацією та фото;
- роль `Роботодавець` з описом вакансії, зарплатою та критеріями пошуку;
- локальне JSON-зберігання в `data/employee.json` і `data/company.json`;
- базовий пошук та ранжування збігів через перетин токенів.

## Швидкий старт

1. Створи та активуй віртуальне середовище:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

2. Встанови залежності:

```bash
pip install -e .
```

3. Створи локальний env-файл:

```bash
cp .env.example .env
```

4. Додай у `.env` значення `BOT_TOKEN`.

5. Запусти бота:

```bash
python main.py
```

## Структура проєкту

- `main.py` - вхідна точка та запуск polling
- `handlers/` - reply, inline та state handlers
- `services/marketplace.py` - доступ до JSON-даних і логіка пошуку
- `formatters/` - форматування карток кандидатів та вакансій
- `keyboards/` - reply/inline клавіатури
- `states/` - FSM-стани для сценаріїв заповнення профілю
- `utils/` - допоміжні функції для роботи з користувачами
- `media/` - локально завантажені фото профілів

## Ідеї для розвитку

### Продукт

- Додати онбординг-квіз після вибору ролі, щоб користувач не губився в меню.
- Зробити окремі картки вакансій за рівнями `Junior`, `Middle`, `Senior`.
- Дати кандидатам кнопку `Відгукнутися`, а роботодавцям - `Запросити на співбесіду`.
- Додати фільтри по зарплаті, ремоуту, країні, стеку і типу зайнятості.
- Вести історію переглядів та обраного: `обране`, `переглянуті`, `приховані`.

### Якість матчингу

- Перейти від простого перетину слів до вагових тегів: `role`, `stack`, `seniority`, `location`.
- Нормалізувати синоніми технологій: `js -> javascript`, `py -> python`, `node -> node.js`.
- Додати антиспам і валідацію введення для зарплат, локацій і стеку.
- Показувати користувачу, чому саме вакансія або кандидат підійшли.

### Технічний розвиток

- Перенести зберігання з JSON у PostgreSQL або SQLite.
- Додати `pydantic`-моделі для анкет і вакансій.
- Винести конфіг у окремий модуль `config.py`.
- Покрити критичну логіку тестами: створення профілю, редагування, матчинг.
- Підготувати деплой на Render, Railway або VPS з webhook-режимом.

## Найближчий практичний крок

Якщо хочеш, наступним повідомленням я можу одразу зробити один із цих варіантів:

1. Перевести проєкт на SQLite без втрати поточної логіки.
2. Додати онбординг і покрокове створення анкети.
3. Покращити алгоритм матчингу кандидатів і вакансій.
